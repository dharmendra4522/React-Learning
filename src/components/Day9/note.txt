This problem is called:
‚ùå Prop Drilling ‚Äì Passing props through components that don‚Äôt need it

üòì Real React Example (Prop Drilling)

function Parent({ user }) {
  return <Child user={user} />;

Parent, Child ko user prop dena pad raha ‚Äî even though unko zarurat nahi.

‚≠ê 3. useContext ‚Äì Step-by-Step with Code

Step 1: Create Context
import { createContext } from "react";

export const UserContext = createContext();

Step 2: Wrap components inside Provider

UserContext.Provider data provide karta hai:

function App() {
  const user = "Dharm sir";

  return (
    <UserContext.Provider value={user}>
      <Parent />
    </UserContext.Provider>
  );
}


Step 3: Consume data using useContext

No need for props.
Direct data le lo:

import { useContext } from "react";

function GrandChild() {
  const user = useContext(UserContext);

  return <h2>Hello {user}</h2>;
}

‚≠ê Important Interview Questions (useContext)
Q1. What problem does useContext solve?

It removes the need to pass props through components that don‚Äôt need them (prop drilling).

Q2. What is Context Provider?

It provides the data to all components below it.

Q3. What is Context Consumer?

A component that uses context data using:

const value = useContext(MyContext);

Q4. Is useContext a replacement for Redux?

For small or medium apps = Yes
For large apps = No (Redux gives more features)

Q5. Does useContext trigger re-renders?

Yes, when Provider value changes ‚Üí consumers re-render.
