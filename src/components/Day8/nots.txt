Q1: Why we call API inside useEffect?

Because React wants side effects (like API calls) to run after the initial render, not during UI creation.

Q2: Why we need loading state?

Without loading, UI blank dikhega.
Loading indicates data aaraha hai.

Q3: Why error handling is important?

Real world API failures common hote hain.
Crash avoid karne ke liye error UI show karna must hai.

Q4: Why async function is created inside useEffect?

Because useEffect ka callback cleanup return expect karta hai â€” async fn return promise karta hai which is not allowed.
